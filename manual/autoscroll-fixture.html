Scrollapp Manual Validation Fixture

Use this page to verify Windows-style middle-click autoscroll behavior against real browser content. It includes plain page scrolling, a nested scroll region, a horizontal overflow strip, and actionable targets that should keep their default behavior instead of starting autoscroll.

How To Use

  1. Open this fixture in a normal browser window.
  2. Start Scrollapp and keep its activation method on plain middle click unless you are testing a different trigger.
  3. Run the matrix below in order so plain body scroll and actionable-target pass-through are checked first.

Recommended Matrix

  • Plain body: middle-click empty body content, then verify only page scroll is latched.
  • Nested container: middle-click inside the inner panel and verify it scrolls without the outer page taking over immediately.
  • Horizontal strip: middle-click the wide strip and move left/right to verify horizontal or diagonal motion.
  • Link target: middle-click the link and verify it follows the anchor instead of starting autoscroll.
  • Button target: middle-click the button and verify the button action fires instead of autoscroll.
  • Text field target: middle-click the text field and verify it does not start autoscroll.
  • Exit swallow: while autoscroll is active, left-click once and verify autoscroll stops without also activating content underneath.
